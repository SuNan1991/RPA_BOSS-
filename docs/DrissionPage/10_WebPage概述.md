# WebPage 概述

来源: https://drissionpage.cn/dp40docs/WebPage/intro

`WebPage` 对象整合了 `SessionPage` 和 `ChromiumPage`，实现了两者之间的互通。

---

## 核心特性

它既可以操控浏览器，也可以收发数据包，并且会在两者之间同步登录信息。

**两种工作模式**:
- **d 模式**: 操控浏览器（与 ChromiumPage 一致）
- **s 模式**: 收发数据包（与 SessionPage 一致）

`WebPage` 可灵活地在两种模式间切换，从而实现一些有趣的用法。

---

## 典型使用场景

### 场景一：复杂登录 + 高效数据采集

网站登录代码非常复杂，用数据包实现过于烧脑。

**解决方案**:
1. 用浏览器（d 模式）处理登录
2. 通过切换模式用收发数据包（s 模式）的方式采集数据

### 场景二：动态内容 + 静态解析

页面内容由 JavaScript 生成，但解析工作量大。

**解决方案**:
1. 用 d 模式让浏览器执行 JavaScript，生成内容
2. 切换到 s 模式或转换元素为 s 模式元素，进行快速解析

## 优势

1. **统一接口**: 两种模式使用逻辑一致，易于上手
2. **灵活切换**: 可根据需要在模式间切换
3. **状态同步**: 自动同步 cookies 和登录信息
4. **性能优化**: d 模式处理复杂交互，s 模式进行快速采集
