# RPA_BOSS 项目脚本说明

本项目提供了 4 个便捷的批处理脚本，用于管理和运行项目。

## 📜 脚本列表

### 1. `kill_all.bat` - 完整清理脚本（推荐）
**功能**:
- ✅ 显示当前运行的进程（Python 和 Node.js）
- ✅ 检查端口占用情况（8000, 8001, 5173）
- ✅ 需要用户确认后才执行清理
- ✅ 清理后验证结果

**使用场景**:
- 首次清理时使用
- 想要了解哪些进程会被清理
- 需要确认清理操作

**运行方式**:
```bash
双击运行 kill_all.bat
```

---

### 2. `kill_all_quick.bat` - 快速清理脚本
**功能**:
- ⚡ 无确认直接清理所有进程
- ⚡ 快速终止 Python 和 Node.js 进程
- ⚡ 强制释放占用的端口

**使用场景**:
- 已确认需要清理，不需要确认
- 快速重启项目
- 端口被占用时的快速修复

**运行方式**:
```bash
双击运行 kill_all_quick.bat
```

---

### 3. `restart_all.bat` - 重启所有服务
**功能**:
- 🔄 自动清理现有进程
- 🔄 启动后端服务（端口 3000）
- 🔄 启动前端服务（端口 5678）
- 🔄 在独立的窗口中运行服务

**使用场景**:
- 完全重启项目
- 确保从干净的状态启动
- 开发环境日常启动

**运行方式**:
```bash
双击运行 restart_all.bat
```

**注意**: 此脚本会自动清理进程，无需先运行 kill_all.bat

---

### 4. `start_all.bat` - 启动所有服务（安全检查）
**功能**:
- ✅ 检查端口是否被占用
- ✅ 如果端口被占用，提示先清理
- ✅ 安全启动后端和前端服务

**使用场景**:
- 已确认没有旧进程运行
- 想要快速启动服务
- 带安全检查的启动

**运行方式**:
```bash
双击运行 start_all.bat
```

**注意**: 如果端口被占用，会提示先运行 kill_all.bat

---

## 🚀 推荐使用流程

### 日常开发启动
```bash
# 方式 1: 一键重启（最简单）
restart_all.bat

# 方式 2: 安全启动
start_all.bat
```

### 遇到端口占用问题
```bash
# 步骤 1: 清理进程（带确认）
kill_all.bat

# 步骤 2: 重新启动
start_all.bat

# 或使用一键重启
restart_all.bat
```

### 快速修复端口问题
```bash
# 快速清理（无确认）
kill_all_quick.bat

# 然后启动
start_all.bat
```

## ⚠️ 常见问题

### 问题 1: 端口被占用
**症状**: 运行脚本时提示端口已被占用

**解决**:
```bash
# 运行清理脚本
kill_all.bat

# 或快速清理
kill_all_quick.bat
```

### 问题 2: Python 命令未找到
**症状**: 运行脚本时提示 'python' 不是内部或外部命令

**解决**:
1. 确认 Python 已安装
2. 将 Python 添加到系统 PATH
3. 或使用完整路径：`C:\Python39\python.exe`

### 问题 3: npm 命令未找到
**症状**: 运行脚本时提示 'npm' 不是内部或外部命令

**解决**:
1. 确认 Node.js 已安装
2. 将 Node.js 添加到系统 PATH
3. 先运行：`cd frontend && npm install`

### 问题 4: 脚本运行后窗口立即关闭
**症状**: 双击脚本后窗口闪退

**解决**:
1. 右键点击脚本
2. 选择"编辑"查看是否有语法错误
3. 或在 CMD 中运行脚本查看错误信息
